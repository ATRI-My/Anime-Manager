# 输入框焦点问题诊断指南

## 问题描述
删除操作后，添加新剧集的弹窗中，输入框显示为灰色，无法点击输入。点击软件窗口外后恢复功能。

## 诊断版本说明
此版本添加了详细的事件日志，用于追踪：
1. 背景遮罩的事件捕获
2. 输入框的事件响应
3. 焦点设置过程

## 测试步骤

### 1. 启动程序
- 双击`启动诊断版.bat`
- 按`F12`打开开发者工具
- 切换到`Console`（控制台）标签页

### 2. 基础测试（无删除操作）
```
1. 进入"写入"板块
2. 选择一个番剧
3. 点击"添加新行"按钮
4. 尝试点击输入框
```

- `EpisodeModal: 模态框打开` - 模态框打开日志

- `=== 输入框 mouseDown 事件 ===` - 输入框鼠标按下事件
- `=== 输入框 click 事件 ===` - 输入框点击事件

### 3. 问题重现测试
```
1. 添加一个剧集并保存
2. 删除该剧集
3. 立即点击"添加新行"按钮
4. 尝试点击输入框
```

**关键观察点**：

- 控制台是否有输入框的事件日志？
- 背景遮罩是否有事件日志？

### 4. 手动诊断命令
在控制台中执行以下命令检查输入框状态：

```javascript
// 查找输入框
const input = document.querySelector('input#number');
if (!input) {
    console.error('未找到输入框');
} else {
    console.log('输入框状态:', {
        id: input.id,
        disabled: input.disabled,
        readOnly: input.readOnly,
        tabIndex: input.tabIndex,
        style: {
            pointerEvents: input.style.pointerEvents,
            display: input.style.display,
            visibility: input.style.visibility
        },
        computedStyle: {
            pointerEvents: window.getComputedStyle(input).pointerEvents,
            display: window.getComputedStyle(input).display,
            visibility: window.getComputedStyle(input).visibility
        }
    });
    
    // 尝试设置焦点
    console.log('尝试设置焦点...');
    input.focus();
    console.log('当前焦点元素:', document.activeElement.tagName, document.activeElement.id);
}
```

## 可能的问题原因

### 1. 事件被阻止
- 背景遮罩捕获了事件
- 父元素有`pointer-events: none`
- 事件冒泡被阻止

### 2. 输入框状态问题
- `disabled`属性为`true`
- `readOnly`属性为`true`
- `tabIndex`为`-1`或无效

### 3. 焦点竞争
- 其他元素保持焦点
- 删除按钮焦点未清除
- 模态框打开时焦点设置失败
