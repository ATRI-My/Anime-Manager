# 模态框关闭问题修复

## 问题描述
在诊断版本中，模态框无法正常关闭。

## 问题原因
在`EpisodeModal.tsx`中，背景遮罩的`onClick`事件被修改为复杂的判断逻辑：
```typescript
onClick={(e) => {
  // 复杂的坐标判断逻辑
  if (clickedInsideModal) {
    console.log('点击在模态框内，不关闭');
    return;  // 这里可能有问题
  }
  onClose();
}}
```

这个判断逻辑可能：
1. 坐标计算不准确
2. 模态框元素查找失败
3. 事件处理逻辑错误

## 修复方案
将背景遮罩的`onClick`事件恢复为简单的关闭逻辑：
```typescript
onClick={onClose}
```

## 修复内容
1. **简化背景遮罩事件**：移除复杂的坐标判断，恢复简单关闭
2. **清理输入框事件**：移除过多的诊断事件监听，减少干扰
3. **保持焦点管理**：保留`EpisodeForm`中的焦点设置逻辑

## 测试方法
1. 运行`启动程序.bat`
2. 进入"写入"板块
3. 点击"添加新行"按钮打开模态框
4. 点击背景遮罩（模态框外灰色区域）
5. **预期结果**：模态框正常关闭

## 文件位置
- 修复版程序：`dist-unpacked-fixed\win-unpacked\`
- 启动脚本：`启动程序.bat`（自动选择修复版）

2. 然后测试删除操作后的输入框问题
