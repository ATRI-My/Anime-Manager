# 项目清理设计方案

## 概述
清理动漫管理项目中的不必要测试脚本和文件，保持项目整洁但不破坏现有工作流程。

## 清理范围
基于用户选择：清理大部分测试文件，保留所有测试代码，合并重复文档。

## 设计方案

### 1. 文件删除
- **删除test-output目录**：测试生成的输出文件
- **删除临时文件**：nul、r.txt
- **删除过时测试脚本**：保留核心脚本，删除重复的run-*.bat变体

### 2. 文档合并
- **合并README文档**：将README-焦点修复版.md、README-未打包程序.md内容合并到README.md
- **保留设计文档**：docs/plans/目录下的设计文档
- **清理测试文档**：test-*.md文件

### 3. 保留内容
- **保留所有测试代码**：.test.ts、.test.tsx、.test.js文件
- **保留核心启动脚本**：启动程序.bat、启动新版.bat、cleanup.bat、generate-unpacked.bat
- **保留示例数据**：example-anime-data.json

### 4. 实施原则
- 不修改任何源代码文件
- 保持git历史完整
- 确保清理后项目仍能正常运行
- 保留有用的调试和测试工具

## 实施步骤
1. 备份当前状态
2. 删除指定文件和目录
3. 合并README文档
4. 验证项目功能
5. 提交清理更改

## 预期结果
- 根目录文件数量减少约40%
- 项目结构更清晰
- 保留所有核心功能和测试能力
- 文档更集中易读
