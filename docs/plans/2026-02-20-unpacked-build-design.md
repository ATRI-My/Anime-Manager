# Anime Manager 未打包版本构建设计

## 概述
为 Anime Manager 桌面应用生成未打包的开发版本，用于测试和验证。

## 需求
- **版本类型**：开发版本
- **目标平台**：Windows
- **功能范围**：完整功能
- **打包形式**：未打包的应用程序目录

## 技术方案
使用 electron-builder 的 `--dir` 模式生成未打包的应用程序。

### 构建流程
1. **代码构建**
   - 编译 TypeScript 主进程代码
   - 构建 React 渲染进程（Vite）
   - 确保所有依赖正确安装

2. **应用程序生成**
   - 使用 electron-builder 生成应用程序目录
   - 包含完整的运行时环境
   - 生成 Windows 可执行文件

3. **验证测试**
   - 检查构建输出完整性
   - 测试应用程序启动
   - 验证核心功能

### 输出结构
```
dist/
├── win-unpacked/                    # 未打包的应用程序目录
│   ├── Anime Manager.exe            # 主可执行文件
│   ├── resources/                   # 应用资源
│   │   ├── app/                     # 应用代码
│   │   │   ├── main/               # 主进程代码
│   │   │   ├── renderer/           # 渲染进程代码
│   │   │   └── ...
│   │   └── ...
│   ├── locales/                     # 本地化文件
│   ├── swiftshader/                 # 图形加速组件
│   └── ...                          # 其他运行时文件
└── ...                              # 其他构建输出
```

## 实施步骤
1. 运行构建命令生成应用程序
2. 验证构建输出完整性
3. 测试应用程序功能
4. 提供使用说明

## 配置说明
使用项目现有的 electron-builder 配置：
- `package.json` 中的 `build` 配置
- `asar: false` 确保未打包
- Windows 目标平台配置

## 成功标准
- 应用程序目录成功生成
- 可执行文件正常启动
- 所有功能正常工作
- 无运行时错误
