# 剧集管理输入框焦点问题分析

## 问题描述
在写入板块中，剧集管理里一旦使用了删除或批量删除功能，再打开添加新行的弹窗输入框就无法点击、输入。但这时点一下软件界面外的任意位置，就会恢复功能。

## 代码分析发现的问题

### 1. EpisodeModal.tsx 中的问题
- **第108-111行**: 模态框内容有 `onClick={(e) => { e.stopPropagation(); }}`
  - 这会阻止点击事件冒泡，可能影响输入框的点击处理
- **第99-101行**: 背景遮罩有 `onMouseDown` 事件处理器
  - `onMouseDown` 在 `onClick` 之前触发，可能捕获了事件
- **第23-66行**: 复杂的强制重绘逻辑
  - 尝试通过修改DOM样式来强制重绘，这可能是问题的征兆而非解决方案

### 2. EpisodeForm.tsx 中的问题
- **第43-143行**: 过度复杂的焦点管理
  - 多次尝试设置焦点，使用 `requestAnimationFrame` 嵌套
  - 尝试强制重绘输入框（第77-79行）
  - 这些代码表明之前已经尝试过修复焦点问题

### 3. WritePage.tsx 中的问题
- **第185行和251行**: 删除操作后调用 `actions.refreshState()`
  - 这会导致组件重新渲染，可能影响焦点状态

## 假设的根因
最可能的问题是：**背景遮罩的 `onMouseDown` 事件处理器在删除操作后仍然保持活动状态，阻止了输入框获得焦点**。

当用户点击软件界面外时，触发了浏览器的焦点恢复机制，从而解决了问题。

## 修复方案
1. 移除背景遮罩的 `onMouseDown` 事件处理器
2. 简化模态框的事件处理逻辑
3. 确保删除操作后模态框能正确重置状态
